## 一、全局架构

### 路由
单页应用，hash 路由：
- `#/dashboard` 仪表盘（默认）
- `#/calendar` 全年日历
- `#/variants` 水果管理
- `#/actions` 行动清单

### 全局布局

```
┌──────────────────────────────────────────────────────────────┐
│ ┌──────────────┬─────────────────────────────────────────┐  │
│ │              │  [Topbar：当前页标题 / 全局操作 / 周次定位] │  │
│ │              ├─────────────────────────────────────────┤  │
│ │   Sidebar    │                                         │  │
│ │   ┌────────┐ │                                         │  │
│ │   │ LOGO   │ │           Main Content                  │  │
│ │   └────────┘ │                                         │  │
│ │              │                                         │  │
│ │   Dashboard  │                                         │  │
│ │   Calendar   │                                         │  │
│ │   Variants   │                                         │  │
│ │   Actions    │                                         │  │
│ │              │                                         │  │
│ │   ────────   │                                         │  │
│ │   导入/导出  │                                         │  │
│ │              │                                         │  │
│ └──────────────┴─────────────────────────────────────────┘  │
└──────────────────────────────────────────────────────────────┘
```

- Sidebar 宽 240px，常驻
- Topbar 高 56px
- 主体最小宽度 980px（桌面优先）
- 配色沿用旧版的绿/金（参考 `index.html` 旧版 `:root` 变量）

### 全局操作按钮（Topbar 右侧）

| 按钮 | 行为 |
|---|---|
| 当前周徽章 | 显示「2026 W23 · 6 月 8-14」，点击跳转到日历当周 |
| 搜索框 | 按变体名 / 标签 / 产区检索，跳转水果管理详情 |
| 导入 JSON | 上传备份覆盖 localStorage |
| 导出 JSON | 下载完整快照（含用户改动 + 行动） |
| 重置初始数据 | 二次确认后清空 localStorage，恢复出厂数据 |

## 二、页面 1：仪表盘 `#/dashboard`

### 信息层级（从上到下）

```
[本周快照]
  ├─ 本周 ISO 周次 / 周区间 / 农历提示（可选）
  ├─ KPI 卡片 ×4
  │   • 本月即上市：N 个变体
  │   • 本月待办行动：N 条（高紧急 X 条）
  │   • 已逾期行动：N 条
  │   • 锁价待签：N 条
  └─ "立即查看"按钮 → 行动清单

[即将上市] 时间窗：未来 4 周
  ┌────────────────────────────────────────┐
  │ 变体名 / 标签 / 进入大量期还有 N 周 / 操作 │
  │ ─────────────────────────────────────  │
  │ 烟台栖霞富士  大宗·稳定  4 周  [行动]    │
  │ 砀山酥梨      大宗·平价  3 周  [行动]    │
  └────────────────────────────────────────┘
  (列出最多 8 条，按时间排序)

[本周行动建议] 来源：自动生成 + 用户手动添加
  - 列出本周内 status=pending 的 Action，按紧急度排序
  - 每条可一键 [标记完成] / [改期] / [跳过]

[已逾期] (折叠卡片，默认收起)
  - status=pending 且 week 已过

[最近变动] (可选，v1.1)
  - 最近 7 天用户编辑过的变体清单
```

### 数据来源
- `variants[]` + `actions[]`
- 自动衍生：「即将上市」= `peakStartWeek` 在未来 4 周内
- 自动衍生：「本周行动」= `actions.where(week == currentWeek && status == pending)`

### 交互
- 卡片均可点击跳转到对应详情
- KPI 数字可点击 → 行动清单页带筛选

## 三、页面 2：全年日历 `#/calendar`

### 核心视图：横向 52 周时间轴

```
                ┌─ 周次轴：W1 W2 W3 ... W52  (固定吸顶)
                │
[分组：仁果类]   │

                │  ↑大量期    ↑价格洼地    ↑稳定
  洛川富士      ████████░░░░░░░░░░░░░░░░████████
  阿克苏冰糖心  ░░██████░░░░░░░░░░░░░░░░░░░░░░░░
梨
  砀山酥梨      ░░░░░░██████████░░░░░░░░░░░░░░██
  库尔勒香梨    ░░░░░░██████████░░░░░░░░░░░░░░██

[分组：核果类]
...
```

### 视觉规则
- 每个变体一行，高 28px
- 横向 52 格，每格宽 18-22px（响应式压缩）
- 色带渲染：
  - **大量期**（peakStart - peakEnd）：实色（按 categoryId 配色）
  - **价格洼地**（lowPrice 区间）：色带顶部多一条金色细线
  - **品质稳定期**（stableQuality）：色带底部多一条绿色细线
  - **最早上市 - 大量期开始** 与 **大量期结束 - 下市**：浅色（30% 透明）
  - **跨年**（peakEnd < peakStart）：色带从 peakStart → 52，再 1 → peakEnd
- 当前周列：红色竖线 + 上方"今"字标
- Hover 单元格：弹出 tooltip 显示该变体的关键事件

### 筛选条（顶部）
- 分组筛选：八大组 toggle
- 进口/国产 toggle
- 优先级筛选：1 / 2 / 3
- 标签 chip：大宗 / 高端 / 礼品 / 错季 / 春节档 / 中秋档 ...
- 排序：按上市期 / 按品类 / 按优先级

### 交互
- 点击变体名 → 水果管理详情侧滑面板
- 点击色带某周 → 弹出"在此周新增行动"对话框
- 顶部"按周/按月/按旬"切换（v1 只做按周，按月按旬 disabled）

```
│     · 洛川富士         │        风险与储存 | 谈判要点 | 行动] │
│     · 阿克苏冰糖心     │                                     │

│ ▶ 柑橘类               │                                     │
│ ...                    │                                     │
│                        │                                     │

└────────────────────────┴─────────────────────────────────────┘
```

### Tab 详情

| Tab | 字段 |
|---|---|
| 基础 | name / variety / origin / originType / tags / 优先级 / 数据可信度 / 备注 |
| 上市期 | season.* 共 8 个周次，配横向滑块或直接输入 1-52 |
| 价格规格 | specs[] + procurementPrice + retailPrice + 损耗 + 毛利 |
| 风险与储存 | risks[] + coldStorage |
| 谈判要点 | negotiationTips[] + priceTrend |
| 行动 | 自动生成 + 手工添加的 actions |

### 交互
- 编辑后实时保存到 localStorage（防抖 1s）
- 顶部显示「已保存于 HH:MM」徽章
- 修改优先级后日历筛选立即响应
- "复制为新变体"用于快速增加同品类新产区

### 交互
- 单条行动可展开查看 checklist
- 点击完成 → status=done, completedAt=now
- 改期 → 弹出周次选择器
- 跳过 → status=skipped + 必填原因
- 批量操作：左侧 checkbox + 顶部"批量完成 / 改期"
- 新增行动按钮 → 选择变体 + 动作模板 + 周次

### 自动生成规则
首次加载或新增变体时，按 `peakStartWeek` + `actionTemplates` 的 `defaultOffsetWeeks` 自动生成 5 条 Action。已生成的不重复生成（用 templateId + variantId + year 去重）。

## 九、技术约束

1. **单 HTML 文件**：所有 CSS/JS 内联，不引外部框架（不用 React/Vue）
2. **原生 JS + 模板字符串**：渲染用 `innerHTML` 或简单的虚拟 DOM 函数
3. **数据加载**：内嵌 `数据_variants.json` 等 JSON 内容到 `<script type="application/json">` 标签
4. **存储**：localStorage key = `procurement-plan-v1`，结构见 `数据_schema.md` 第六节
5. **浏览器兼容**：macOS Safari 17+ / Chrome 120+，不必兼容 IE
6. **打开方式**：本地 `file://` 双击可用，无需 server

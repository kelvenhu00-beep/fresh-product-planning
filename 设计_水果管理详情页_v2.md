# 水果管理详情页 v2 设计稿（展示优先 + 可视化）

> 维护：Claude
> 用途：替换 `设计_页面布局.md` 第四节「水果管理」的现有方案。
> 触发：v1 实现的详情页全是输入框，缺少可视化、信息密度不够专业。

---

## 一、核心思路

| 维度 | v1 现状 | v2 目标 |
|---|---|---|
| 默认状态 | 全表单（输入态） | 展示卡（阅读态） |
| 视觉密度 | 纯文字、纯表格 | 卡片 + 徽章 + 迷你图表 + 时间轴 |
| 编辑入口 | 直接改值 | 每 section 右上「编辑」按钮，切换为表单 |
| 信息层次 | Tab 切换 | 全长滚动页，区块化展示 |

新设计像一份**采购品类档案**：上层一眼看全貌，下钻可以编辑。

---

## 二、整体布局

仍是左右两栏。左侧品类树不变。右侧详情区改为**单页滚动**，不再用 Tab：

```
┌─────────────────────────────────────────────────────────────┐
│ ▌左栏：品类树（不变）│ ▌右栏：变体详情（全长滚动）              │
│                       │                                         │
│   ▼ 仁果类             │  [Section H：变体头部]                  │
│     ▼ 苹果 (6)         │  [Section A：上市周期时间轴]             │
│       · 烟台栖霞富士   │  [Section B：规格与价格]                 │
│       · 洛川富士        │  [Section C：风险与储存]                 │
│       ...              │  [Section D：谈判要点]                  │
│   ▶ 核果类             │  [Section E：行动计划]                  │
│   ...                 │  [Section F：原始字段（折叠，编辑兜底）]  │
└─────────────────────────────────────────────────────────────┘
```

每个 Section 右上角有一个轻量「编辑」按钮（笔形 icon + 字「编辑」），点击切换为该区块的表单。

## 三、各 Section 详细设计

### Section H · 变体头部

```
┌─────────────────────────────────────────────────────────────┐
│  仁果类  ›  苹果  ›  烟台栖霞富士          [复制] [删除] [⋯]   │
│                                                              │
│  烟台栖霞富士                                                │
│  红富士  ·  山东 · 烟台 · 栖霞                                │
│                                                              │
│  [国产] [优先级 1] [数据可信度 高] [大宗] [稳定] [高出货]       │
└─────────────────────────────────────────────────────────────┘
```

- 大字号变体名（28px）
- 品种 + 产地用一行展示（中等字号、灰色）
- 徽章组（chips）：originType（国产=绿底/进口=蓝底/混合=灰底） + 优先级（1=红 / 2=金 / 3=灰） + 数据可信度（颜色对应等级） + tags
- 右上轻量操作按钮（复制 / 删除 / 更多）

### Section A · 上市周期时间轴（核心可视化）

```
┌─ 上市周期 ──────────────────────────────────── [编辑] ┐
│                                                      │
│  W1   W10   W20   W30   W40   W50                    │
│  │     │     │     │     │     │                     │
│  ▓▓░░░░░░░░░░░░░░░░░░░░░░▒▒▒▒▒█████▓▓▓▓▒▒▒▒          │
│        ↑                                              │
│      今 W23                                           │
│                                                      │
│   🌱 最早上市：W40 · 10 月初                          │
│   🔥 大量上市：W43 - W8（跨年）· 10 月下 - 2 月底     │
│   💰 价格洼地：W45 - W3 · 11 月 - 1 月                │
│   ⭐ 品质稳定：W43 - W12 · 10 月下 - 3 月             │
│   🌙 下市：W22 · 5 月底                               │
└──────────────────────────────────────────────────────┘
```

- 顶部一条 52 格的迷你时间轴
  - 高 18px，每格 ~1/52 宽
  - 颜色填充按 5 个阶段：
    - 最早上市 → 大量开始：浅色（30% 透明）
    - 大量期：实色（按品类组色）
    - 大量结束 → 下市：浅色（30% 透明）
    - 价格洼地：上方覆盖一条金色细线
    - 品质稳定：下方覆盖一条绿色细线
  - 当前周：红色竖线 + 「今 W?」标注
- 下方文字摘要 5 行，每行带 emoji 图标 + 周次 + 月份折算

```
│  ┌──┐ 70# 70-75mm 普通装│ │ 采购                                 │
│  ┌──┐ 75# 75-80mm 商超主│ │ ▌▌▌▌▌▌▌▌▌▌▌▌▌▌▌▌▌▌▌▌▌                │
│  ┌──┐ 80# 80-85mm 礼盒主│ │ 2.5 ━━━━ 3.8 ━━━━━━━━━━━━━━ 5.5      │
│  ┌──┐ 85# 85mm+ 高端礼盒│ │                                      │
└────────────────────────┘ │ 零售                                  │
                            │ ▌▌▌▌▌▌▌▌▌▌▌▌▌▌▌▌▌▌▌▌▌▌▌▌▌▌▌▌▌▌▌▌▌▌  │
┌─ 毛利与损耗 ────────────┐ │ 5.0 ━━━━ 7.0 ━━━━━━━━━━━━━━ 10.0     │
│ 毛利率（典型 35%）       │ │                                      │

│ ▓ 占比小                 │

```

- **规格分级**：每个 spec 一张紧凑卡，grade 大号、description 小号
- **价格区间**：采购价和零售价分别画水平条形图
  - 左端 low、中段 typical（深色凸起）、右端 high
  - 数字标注三个点
  - 单位放右上角
- **毛利率**：一条进度条 + 数字范围
- **损耗**：两个小数字 + 占比说明

## 四、编辑模式交互

两种入口：

1. **Section 内联编辑**：点 section 右上「编辑」 → 该 section 切换为表单状态 → 「保存」/「取消」 → 切回展示
2. **完整字段编辑**：Section F 折叠面板 → 展开即 v1 的所有表单 Tab

实现侧推荐：
- 每个 section 一个 React-like 状态 `editingSection: 'season'|'price'|'risk'|...|null`
- 切换时只重渲染该 section
- 保存路径走 `scheduleSave` + 「已自动保存」toast（已有逻辑）

## 五、新增 CSS 组件（给 Codex 参考）

| 组件 | 用途 | 实现要点 |
|---|---|---|
| `.detail-header` | 变体头部块 | flex 布局，大字号名称 |
| `.chip` | 各种徽章 | 已有 `.badge`，复用并扩展颜色类 |
| `.timeline-strip` | 52 周迷你时间轴 | 横向 grid，52 列；阶段用 5 种 class 控制 |
| `.timeline-summary` | 时间轴下方 5 行说明 | flex column + emoji + 周次 + 月份 |
| `.spec-card` | 规格卡 | 小卡片，grade 大字 + desc 小字 |
| `.price-bar` | 价格条形图 | 三段水平 bar，low / typical（深）/ high |
| `.donut` 或 `.gauge` | 毛利率 | 简化用 progress bar 即可 |
| `.risk-card` | 风险卡片 | 左侧色块（按类型）+ icon + 描述 |
| `.storage-info` | 冷藏信息块 | grid 4 个属性，带 icon |
| `.tip-list` | 谈判要点 | 编号圆点 |
| `.trend-timeline` | 价格趋势时间线 | 竖向 list |
| `.action-timeline` | 行动时间线 | 横向节点 + 连线 |
| `.collapse-card` | 折叠面板 | 用 `<details>` 元素即可 |

颜色复用 v1 的 `:root` 变量。对于阶段着色，新增：

```css
--phase-pre: rgba(47,111,79,.25);    /* 浅绿 */
--phase-peak: var(--green);          /* 实色 */
--phase-post: rgba(183,121,31,.25);  /* 浅金 */
--phase-low: var(--gold);            /* 价格洼地条纹 */
--phase-stable: var(--green);        /* 品质稳定条纹 */
```

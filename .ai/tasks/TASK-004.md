## 背景

当前 `#/festivals` 节庆档的所有节庆周次是硬编码的（如春节 = W4-W8、端午 = W23-W24），
每年农历节日实际落点不同，2027 年以后这些数字就会偏差越来越大，
导致「提前 4 周锁货」提醒和「当前周高亮」功能不准确。

需要替换为基于当前年份的动态计算。

## 目标

节庆档的每个农历节日（春节、元宵、清明、端午、中秋），其窗口期每年自动根据当年实际日期计算，
公历节日（五一、国庆、双11、双12、圣诞、元旦）直接由公历推算即可。

## 技术约束

- **框架/库**：零依赖，纯 JS 实现，不引入任何外部农历库
- **不能动的文件**：`procurement_server.py`、多设备同步逻辑、其他页面
- **精度要求**：ISO 周编号正确即可（± 1 周以内）
- **年份范围**：至少支持 2025–2035

## 实现方案

### 方案：查表 + 公历推算（推荐）

农历节日用预埋查表（精确），公历节日直接公式推算（无误差）。

#### 公历节日（直接由年份推算）
```js
// 元旦 = Jan 1
// 清明 = 4月4日或5日（用蔡勒公式：(y*5/4|0) + 132 - (y/100|0) 取模3...
// 简化处理：2025-2035 清明均为 4/4 或 4/5，用以下公式
// qingming(year) = year <= 2044 ? '04-04' : '04-05'  (实测范围内均为 4/4 或 4/5)
// 五一 = May 1
// 国庆 = Oct 1
// 双11 = Nov 11, 双12 = Dec 12, 圣诞 = Dec 25
```

### 核心辅助函数

```js
// 日期字符串 → ISO 周编号
function dateToIsoWeek(dateStr) {
  const d = new Date(dateStr);
  const jan4 = new Date(d.getFullYear(), 0, 4);
  const dayOfYear = Math.floor((d - new Date(d.getFullYear(),0,0)) / 86400000);
  const woy = Math.ceil((dayOfYear + jan4.getDay()) / 7);
  return woy;
}

// 日期偏移 N 天
function addDays(dateStr, n) {
  const d = new Date(dateStr);
  d.setDate(d.getDate() + n);
  return d.toISOString().slice(0,10);
}
```

### 节庆定义函数 `getFestivals(year)`

返回与当前 festivals 页期望格式一致的数组，每项包含：
`{ id, name, emoji, weeks: [startWeek, endWeek], windowLabel, desc }`

| 节庆 | 计算方式 | 窗口宽度 |
|---|---|---|
| 元旦 | Jan 1 → ISO week | ±1 周（W52-W1） |
| 春节 | SPRING_FESTIVAL[year] | 春节前 1 周 ~ 春节后 2 周 |
| 元宵 | 春节 + 14 天 | ±1 周 |
| 清明 | Apr 4 | ±1 周 |
| 五一 | May 1 | May 1 前 1 周 ~ 五一后 1 周 |
| 端午 | DRAGON_BOAT[year] | ±1 周 |
| 中秋 | MID_AUTUMN[year] | 中秋前 1 周 ~ 中秋后 1 周 |
| 国庆 | Oct 1 | Oct 1 前 1 周 ~ 后 1 周 |
| 双11 | Nov 11 | ±1 周 |
| 双12 | Dec 12 | ±1 周 |
| 圣诞 | Dec 25 | ±1 周 |

## 潜在风险 / 踩坑提示

1. **ISO 周跨年**：`dateToIsoWeek` 要处理 12/28-31 可能属于下一年 W1，
   以及 1/1-3 可能属于上一年 W52/W53 的情况。
   建议直接用标准 ISO 8601 算法，或复用现有 `isoWeek()` 函数（先 grep 确认函数名）。

2. **查表缺失年份**：若 `year` 不在查表范围，降级到最近年份的偏移估算，
   不要直接报错或返回 undefined。建议 fallback: `SPRING_FESTIVAL[year] ?? SPRING_FESTIVAL[2035]`。

3. **现有节庆格式**：替换前先 grep 确认现有 festivals 数据结构，
   确保 `getFestivals()` 返回的字段名和格式与渲染函数期望的完全一致，不要改渲染函数。

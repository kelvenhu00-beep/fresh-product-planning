- ✅ 关键决策落定（业务场景 / 优先级 / 进口品归属）

### Claude 接下来要做（按顺序）

2. **填充 30 个高优先级变体** —— 输出 `数据_variants.json`
3. **标准动作模板** —— 输出 `数据_actionTemplates.json`
4. **四页 wireframe** —— 输出 `设计_页面布局.md`
5. 等待 Codex 实现完毕后审阅效果

### Codex 接下来要做（可并行，不必等 Claude 全部完成）
1. **环境就位**：单 HTML 文件骨架（不依赖任何 npm 包，纯 HTML+JS+CSS）
2. **读 Claude 的 schema** → 设计 localStorage 数据结构
3. **实现四个页面**（按 `设计_页面布局.md`，未交付前先用占位）：
   - 仪表盘
   - 全年日历（52 周横向时间轴，按变体着色）
   - 水果管理（品类-变体两层，CRUD）
   - 行动清单（按时间排序，可标记完成）
4. **数据加载**：内嵌 `数据_variants.json` 作为初始数据，localStorage 缓存用户修改
5. **导入/导出 JSON**：与旧问卷的 JSON 格式无关，定义新格式
6. **本地预览验证**：确保 macOS Safari + Chrome 双击 `index.html` 可正常运行
7. **完成后在 `工作日志.md` 写入交付清单**

### 接口约定（Codex 读 Claude 产物）
- 所有数据文件以 UTF-8 JSON 格式存放
- 字段命名遵循 `数据_schema.md` 中定义的 key（驼峰，英文）
- 日期统一用 ISO 周编号（1-52，跨年用 53）
- 价格单位统一标注 `unit` 字段，不假定

## 四、用户介入的最低阈值

只有以下情况需要打断用户：

1. **业务方向有根本分歧** —— 两个 AI 各执一词无法收敛时
2. **真实业务数据缺失** —— 需要用户提供真实历史价格、供应商名单、内部数据
3. **v1 完成后的验收** —— 用户试用并反馈

其他情况下，按本协议自行决策、推进、交付。
